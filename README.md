# ddnet_int128

坐标使用 **int128** 的 DDNet 魔改。

## 改了啥

- 世界坐标 / 物理等走 **int128**（固定点 / 大整数像素），不再卡在普通 `int` / `float` 那点点范围。
- **网络协议** 已改（角色位置等按多 limb / i128 传），客户端和服务器要配套用本仓库构建，别硬连原版 DDNet 协议。
- 其它：大坐标传送、碰撞边缘、HUD 显示等按大坐标场景修过。

## 渲染（尽力了）

渲染这边 float 精度差不多到 **≈ 33554430 格**（约 `2^25`）就会顶不住，远处画面会烂掉 / 断掉。  
尽力修过相对原点、EXTEND 边砖之类，**甚至没有「原版 DDNet 那种感觉」走得那么顺**，只能说凑合能看一段。

**但是物理坐标可以去很远。** 至少你可以去：

```text
5316911983139663491615228241121378303
```

xD

## 防止未响应（client）

大坐标瞬移 / 镜头乱跳时客户端可能卡死，请开：

```text
cl_protect_large_move
```

## 说明

- 本仓库基于 [ddnet/ddnet](https://github.com/ddnet/ddnet) 改。
- 实验性质，协议不兼容原版；出图外碰撞/渲染行为以当前代码为准。
- 构建方式大体同上游，见 `docs/BUILDING.md`。
